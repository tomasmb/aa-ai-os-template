# digests/email-weekly.md — Weekly owner digest template

> Template for the weekly email digest sent every Friday by the assistant.
> Composed locally (no external service). Delivered via the user's default
> mail client (primary) or SMTP relay (opt-in).
>
> The assistant fills the placeholders, summarizes per the guidelines in
> `packs/company-rituals.md` (Ritual 3), and hands the final draft to the
> user for a one-click send — or sends automatically if the user has opted
> into `digests.smtp_enabled`.

## Email shape

- **To:** the user (always). Optionally CC the user's manager if
  `digests.digest_cc_manager = true` in `WORKSTYLE.md`.
- **From:** the user's own email address. The assistant never impersonates.
- **Subject:** `Alpha weekly — week of <Monday date> — <user first name>`
- **Body:** the template below, sections omitted when empty.

## Body template

```text
Hi <user first name>,

Week of <Monday YYYY-MM-DD> — here's what your brain surfaced.

## Headline
<One sentence on the week's biggest move. Pulled from the week's highest-
confidence Decision owned by the user OR the most-updated Project the
user owns. Never two sentences.>

## Projects you own (<N>)
- **<Project name>** — <Status>. <Next milestone or blocker in one line.>
  <Notion link>
- **<Project name>** — <Status>. <Next milestone or blocker in one line.>
  <Notion link>

## Decisions made this week (<N>)
- **<Decision title>** (<Decided on>) — <One-line outcome.> <Notion link>

## Insights worth your attention (<N>)
- **<Insight title>** (Strength: <strong/medium/weak>, Surface count: <N>)
  — <Suggested action, if any.> <Notion link>

## Goals you own (<N>)
- **<Goal>** (<Period>) — <Status>. <One-line progress note or next KR.>
  <Notion link>

## Meetings this week (<N>, top 5)
- **<Meeting title>** (<Date>) — <Kind>, <N> attendees. <Decisions: N,
  Insights: N>. <Notes link>

## Waiting on you
- <Name> — <what they're waiting on, one line> (surfaced <date>).
- <Name> — <what they're waiting on, one line> (surfaced <date>).

## This week's open threads going into next week
- <Thread 1 — one line, ends with the concrete next step.>
- <Thread 2 — one line, ends with the concrete next step.>

—
Auto-generated by your Alpha assistant. Reply to this email with
*"skip next week"* to pause one digest, or *"stop digests"* to turn them off.
```

## Section rules

| Section | Include if | Omit if |
|---|---|---|
| Headline | always | one sentence only, never omit |
| Projects you own | user has ≥1 `🚀 Projects` row with `Owner = user, Status != Done` | zero projects — omit the section |
| Decisions made | user has ≥1 `✅ Decisions` row with `Owner/Participants includes user, Decided on in the last 7 days` | none — omit |
| Insights | user has ≥1 `💡 Insights` row with `Related people includes user, Suggested action != empty` | none — omit |
| Goals you own | user has ≥1 `🎯 Goals` row with `Owner = user, Status in {committed, on_track, at_risk, missed, achieved}` | zero goals — omit |
| Meetings this week | user has ≥1 `🗓 Meetings` row with `Date in last 7 days, Attendees includes user` | zero meetings — omit. Cap at top 5 by decisions/insights produced. |
| Waiting on you | `memory/recurring-work.md` has entries tagged `waiting_on_<user>` | empty — omit |
| Open threads | today's daily note + recent daily notes carry unresolved items | clean slate — omit |

**If every section is empty → send nothing.** The assistant reports
in-chat: *"Quiet week for you — no digest to send."*

## Length budget

- Headline: 1 sentence.
- Each list section: cap at **5 items**. More than 5 → keep top 5 by
  recency + confidence + user-ownership, add a line: *"(+ 3 more in your
  brain — ask me anytime.)"*
- Total email length: aim for <300 words. Never exceed 500.

## Sensitivity filtering

Every candidate line runs through the Rule 14 sensitivity classifier before
inclusion. Items flagged sensitive are excluded silently, no footnote (that
would leak the fact that something sensitive exists).

Items the user has marked *"that was private"* or *"forget that"* are
**never** included — these are queried out at the source.

## Delivery

Two paths — the assistant picks based on `WORKSTYLE.md`:

### Default path — user's default mail client

1. Compose the full email body per the template above.
2. Generate a `mailto:` URL with pre-filled subject + body:
   `mailto:<user-email>?subject=<url-encoded>&body=<url-encoded>`
3. Present to the user in-chat: *"Weekly digest ready. Opening your mail
   now — review and send."*
4. Trigger `open <mailto-url>` (macOS), `xdg-open` (Linux), `start`
   (Windows). The user's default mail client opens with the draft.
5. User hits send. Done.

### Opt-in path — SMTP relay

If the user has set `digests.smtp_enabled = true` in `WORKSTYLE.md` and
provided credentials in `logs/.smtp-credentials.env` (gitignored):

1. Compose the email body.
2. Send via `sendmail` / SMTP with the configured credentials.
3. Report in-chat: *"Weekly digest sent to <your email>. Anything you want
   to follow up on?"*

SMTP is **off by default**. Most users prefer the one-click default client
flow.

## What NEVER goes in a digest

- Sensitive content (Rule 14 categories 1–8).
- Content from other users' local memory (not reachable anyway — each
  user's assistant only reads its own folder).
- Raw transcripts or direct quotes from meetings (summarize instead).
- Brain rows with `Confidence = low` unless explicitly confirmed.
- Items the user has flagged private, forgotten, or skipped.
- Mentions of the folder, files, MCPs, Notion API, or versioning (Rule 1).

## Files this template depends on

- `packs/company-rituals.md` — invokes this template
- `packs/company-brain.md` — brain queries
- `WORKSTYLE.md` — delivery config
- `USER.md` — user's email address
- `memory/recurring-work.md` — "waiting on you" source
- today's and the past 7 days' daily notes — open threads
